Block-builder pipeline — how the deep-dive decomposes

The map of the v2 deep-dive. Retrieval (plan_trim_select) lands two claim ledgers; the A1 (surface-evidence) and A2 (surface expression) builders each extract one structured block from their slice. 10 builders total — 3 A1, 6 A2, 1 cross-focus — of which three are ROLLUP steps (evidence_grade, biological_context_grade, and the synthesizer) that aggregate across blocks rather than extract a single observation type. The cross-focus SHARED risks_builder consumes the MERGED A1+A2 ledger plus the deterministic ECD / homo-oligomer signals, so it runs AFTER the fan-out (not on one focus's slice) and produces the canonical accessibility_risks block the synthesizer used to author. Use this to judge whether the split into per-observation builders + rollups hangs together.

  • filters_llm (4 LLM rollup chips: expression_level/breadth, surface_specificity, has_known_ligand)
  • accessibility_risks (per-risk present/severity/cited_evidence_ids)
  • confidence + confidence_reasoning (anchors on the triage prior + deterministic data)

6 — post-passes (deterministic)

orchestrator post-pass helpers (no LLM)

Input

  • SynthesizerDraft (B’s output above)
  • DeterministicFeatures (from stage 3)
  • IdentifierBundle (curator family tags)

Process

Three post-passes: scrub_headline_risks deletes structurally-incoherent risks, _attach_deterministic_families overwrites hgnc_gene_groups + uniprot_family on the executive_summary, secreted_form_post_pass upgrades the secreted_form risk from isoform topology if the synthesizer hadn’t (this is now defense-in-depth since B sees the topology at stage 5; was load-bearing before this PR). Then _derive_filters composes the full 17-field Filters block from the LLM rollups + the deterministic blocks.

Output

  • SurfaceomeRecord (gene, executive_summary, filters, surface_evidence, biological_context, accessibility_risks, deterministic_features, evidence, knowledge_gaps, …)
